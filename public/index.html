Congratulations, you've put PHP on Trax!

Before you move on, verify that the following conditions have been met:

  1. The log and public directories must be writable to the web server (chmod -R 775 log and chmod -R 775 public).
  2. Trax on Apache needs to have mod_rewrite enabled.
    Somewhere in your httpd.conf, you should have:
    LoadModule rewrite_module libexec/httpd/mod_rewrite.so
    AddModule mod_rewrite.c
  3. Edit public_html|public/.htaccess and add to the end of the include_path the path to the trax/config dir
    example: php_value include_path .:/usr/local/lib/php:/home/myaccount/config

Take the following steps to get started:

  1. Create empty development and test databases for your application.
    Recommendation: Use *_development and *_test names, such as database_development and database_test
  2. Edit config/database.ini with your database settings.
  3. Create controllers and models using the generator in script/generate.php
    Help: Run the generator with no arguments for documentation or with only model or controller for each instructions
  4. main site layouts should be in app/views/layouts
    default: public.phtml
    content from view files are displayed in the layout phtml file
    by using the variable $content_for_layout
  5. Develop your Trax application!

Trying to setup a default page for Trax using Routes? You'll have to delete this file (public/index.html) to get under way. Then define a new route in trax/config/routes.php of the form: 

    # this is for the root directory
    $route->connect( "", array(":controller" => "welcome") );
    # this is the default route
    $router->connect( ":controller/:action/:id" );

Having problems getting up and running? First try debugging it yourself by looking at the php error log files on your server.
