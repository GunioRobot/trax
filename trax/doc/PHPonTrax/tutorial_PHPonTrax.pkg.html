Getting Started With Trax

Installing Trax In Your Home Directory

In the steps below, we will install Trax in the home directory of user haas and verify basic functions. When following these instructions, substitute your own username for haas.

Trax requires PHP 5, the Apache web server and a relational DBMS such as MySQL. Trax also requires several PEAR modules. The examples below use Fedora Core 4 Linux, and may need adjustment for other systems. The installation procedure below begins by checking for the required software.

  1. Verify that PHP 5 is available from the command line. On the command line, type: 
    $ php --version
PHP 5.0.4 (cli) (built: Nov  8 2005 08:27:12)
...
    You need to have at least version 5.0.3??? of PHP.
  2. Trax needs the DB, Mail and Mail_mime PEAR modules. Most current Linux distributions come with PEAR installed. To test your system, type: 
    $ pear list
    1. If the PEAR library is installed you will see a list of installed packages: 
      Installed packages:
===================
Package              Version  State
...
DB                   1.7.6    stable
...
Mail                 1.1.4    stable
Mail_Mime            1.3.1    stable
      If any of the required packages are missing, you need to install them. Become root, then use the PEAR install command to install whichever packages are missing. For example, to install DB: 
      $ su -
Password: ...
# pear install DB
downloading DB-1.7.6.tgz ...
Starting to download DB-1.7.6.tgz (124,807 bytes)
............................done: 124,807 bytes
install ok: DB 1.7.6
    2. If PEAR is not present you will get an error message similar to "pear: command not found". In this case you need to install PEAR. You can find the necessary code on the Trax download page. Download and install PEAR according to the instructions there. The PHP_LIB_ROOT constant mentioned is in config/environment.php.
    Optionally, install PHPUnit2 to run the Trax regression tests, and phpDocumentor to regenerate this documentation.
  3. Check your Apache configuration. Verify that Apache includes the mod_php, mod_rewrite and mod_userdir modules. To check, examine file /etc/httpd/conf/httpd.conf and /etc/httpd/conf.d/php.conf for the lines: 
    LoadModule userdir_module modules/mod_userdir.so
LoadModule rewrite_module modules/mod_rewrite.so
LoadModule php5_module modules/libphp5.so
AddHandler php5-script .php
AddType text/html .php
    If any of these modules is missing or disabled, you need to add it. NOTE: it is possible that these modules have been compiled into your version of Apache.
  4. Verify that your home directory is accessible to Apache. Examine file httpd.conf for something like: 
    UserDir public_html
    If UserDir is disable, you need to change the configuration file to set UserDir to some appropriate subdirectory (public_html is customary). If your configuration sets UserDir to some other subdirectory of your home directory, make a note of the value.
  5. Verify the name of the per-directory configuration file. File httpd.conf should contain a line like that below: 
    AccessFileName .htaccess
    If AccessFileName is not .htaccess, use the filename in the configuration instead of .htaccess in the steps below, and rename file ~/public_html/.htaccess to the name you found in the configuration.
  6. Verify that Apache will follow commands from a .htaccess file in subdirectory public_html. File httpd.conf should contain lines like those below. If UserDir is not public_html, then the Directory line below should match the subdirectory specified in UserDir. File .htaccess must be given at least the override permissions below in order for mod_rewrite to work. 
    <Directory /home/*/public_html>
    AllowOverride FileInfo Options
    Options FollowSymLinks
</Directory>
  7. If any of the steps above produced a change to the Apache configuration, you must restart Apache: 
    $ su -
Password: ...
# apachectl restart
  8. Verify that your home directory doesn't have anything valuable in subdirectories public or trax.
  9. Download the Trax distribution into your home directory and untar it.
  10. If the value assigned to UserDir in httpd.conf is other than public, rename directory public to the value assigned to UserDir: 
    $ mv public public_html
  11. As you work with Trax, you may find it helpful to monitor the Apache error log. You can do this with less. Type: 
    $ su -
Password: ...
# less /var/log/httpd/error_log
    When less starts, give it the command F to start continuous monitoring of file error_log.
  12. Browse to http://localhost/~haas/index.html (substitute your username for haas) and verify that you see a page that begins Congratulations, you've put PHP on Trax!. At this point you know that Apache will access a static HTML file untarred from the Trax distribution in your home directory.
  13. You can verify that PHP works in your public_html directory and is the right revision by creating a file that runs phpinfo(): 
    $ cd ~/public_html
$ cat > phpinfo.php
<?php phpinfo() ?>
    Then browse to http://localhost/~haas/phpinfo.php. You should see a page that starts PHP Version ... and details how PHP was built. You need to have at least version 5.0.3??? of PHP. At this point you know that Apache will execute a simple PHP script from your home directory.
  14. With your favorite editor, edit file ~/public_html/.htaccess to change the include_path line so that it includes the correct location of the Trax configuration directory in your home directory, plus any other directories currently searched by PHP, particularly the PEAR library. To find the current value of include_path, check the PHP configuration file /etc/php.ini: 
    $ grep include_path /etc/php.ini
...
include_path = ".:/usr/share/pear"
...
    You may also see other lines returned by grep. Lines that begin with ; are comments and don't affect the running PHP configuration. The information you want is on the line that does not start with ;. Add the include_path information from/etc/php.ini to .htaccess: 
    php_value include_path .:/usr/share/pear:/home/haas/trax/config
    In the same file, add a prefix before the / from dispatch.php in the last RewriteRule line. Change: 
    RewriteRule ^(.*)$ /dispatch.php?$1 [QSA,L]
    To: 
    RewriteRule ^(.*)$ /~haas/dispatch.php?$1 [QSA,L]
  15. Browse to http://localhost/~haas/dispatch.php and verify that you see a page that says: Controller ~haas not found??? If you don't get this page and the Apache error log shows a message such as "Failed opening required 'environment.php'" then there is something wrong with the include_path value in .htaccess. If you get the expected page, you know that the include_path includes the Trax configuration.
  16. Delete file index.html from ~/public_html: 
    $ rm ~/public_html/index.html
  17. Browse to http://localhost/~haas/ and verify that you see the same page as when you browsed directly to dispatch.php, above. If you get the expected page, you know that your mod_rewrite configuration works correctly.
  18. With your favorite editor, open file ~/trax/config/environment.php and change the line that defines TRAX_URL_PREFIX to: 
    define("TRAX_URL_PREFIX", "~haas");
    Also edit the line that defines TRAX_PUBLIC to change public to public_html.
  19. Change file permissions on the log files in ~/trax/log so that Apache can write to them: 
    $ su -
Password: ...
# chmod 777 /home/haas/trax/log/*
You now have a working Trax installation in your home directory.

Defines

All the definitions below are set in file config/environment.php
  • DEBUG
    boolean set to true if TRAX_MODE is 'development', false otherwise. Determines whether PHP error messages should be sent to the browser; true means show error messages on the browser as well as the error log, false means send error messages to only the error log. Implemented by calling ini_set() at the time DEBUG is defined.
  • PHP_LIB_ROOT
    Directory containing PHP libraries. This directory will be added to the PHP include path.
  • TRAX_LIB_ROOT
    Directory containing Trax libraries. Set to a subdirectory of TRAX_ROOT or PHP_LIB_ROOT and added to the PHP include path.
  • TRAX_MODE
    Contains one of 'test', 'development' or 'production'. Determines which database to use. Get it from $_SERVER['TRAX_MODE'] if that exists
  • TRAX_PATH_SEPERATOR
    Character to use as the separator when defining the PHP include path. ';' for Windows, otherwise ':'.
  • TRAX_PUBLIC
    Subdirectory of the user's home directory referenced by the Apache configuration variable UserDir.
  • TRAX_ROOT
    Filesystem path to the top of the Trax file tree.
  • TRAX_URL_PREFIX
    That part of a URL which refers to a Trax application that comes after the domain name and before the controller. Usually empty. In the case of a Trax application in the home directory of user username, TRAX_URL_PREFIX would be set to '~username'.
  • TRAX_VIEWS_EXTENTION
    File extension for views, default 'phtml'.

Globals

  • $GLOBALS['ACTIVE_RECORD_DB']
    Database connection resource for the currently open database connection, or PEAR or DB error if the connection attempt failed. Set in ActiveRecord::establish_connection().
  • $GLOBALS['current_action_name']
    Name of action parsed from the URL. Set in ActionController::process_route() Value stored before knowing whether the method for this action actually exists in the relevant controller.
  • $GLOBALS['current_controller_name']
    Name of the controller as parsed from the URL. The name is in lower case without the "_controller" suffix. Set in ActionController::process_route(). Not set until the controller PHP file has been opened and the controller object created successfully.
  • $GLOBALS['current_controller_object']
    Successfully created controller object. Set in ActionController::process_route()
  • $GLOBALS['current_controller_path']
    Controller URL path below TRAX_URL_PREFIX. Set in ActionController::process_route(). Not set until the controller PHP file has been opened and the controller object created successfully.
  • $GLOBALS['TRAX_DB_SETTINGS']
    Array with keys 'test', 'development' and 'production' specifying the database connection parameters for each of the three modes of operation. Set in file config/environment.php from values parsed from file database.ini. Used when opening a connection to the database.
  • $GLOBALS['TRAX_INCLUDES']
    Array with keys "models", "views", "controllers", "helpers", "layouts", "config", "environments", "lib", "app", "log" and "vendor". Values are the subdirectories of TRAX_ROOT where Trax files of the corresponding category are stored. Used to create filesystem paths to access Trax files. Set in file config/environment.php
